The one rule: one brain, one engine. Flowise orchestrates; Python computes. Never re-implement the reasoning inside Flowise, and don't let the old chat and the Flowise agent answer independently — they'd drift. Because Flowise calls the same engine, both always give the same numbers.

Safe transition (no big-bang)

1. Keep the current app running (it's the demo).   2. Stand up Flowise alongside, calling the same tool endpoints.   3. When the Flowise agent matches the current chat, retire /api/query and point the frontend at Flowise. The engine/tools never change through any of it.
