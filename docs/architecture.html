ATS Engineering Assistant

How the system works

From a client requirement to a traceable engineering specification — and the roadmap beyond it.

Understands the request

A local language model reads the enquiry and extracts the equipment type and sizing inputs. Casual chat and “how many offers…” questions are answered directly — the counts are computed in code, not guessed.

One resolver, two modes

A single engine builds the specification. A policy decides the source of truth: Consulting uses engineering knowledge; ATS (say “refer db”) builds from your historical offers. Same schema, different evidence.

Every number is traceable

Each value is tagged with its origin (client-given, engineering rule, or historical consensus), sanity-checked against engineering limits, and given a confidence score — with the source offer cited.

FastAPI · ChromaDB · MiniLM embeddings · Rule Engine (Pydantic) · Ollama Qwen2.5 · React  •  Human-in-the-loop: outputs are engineer-reviewed drafts.
